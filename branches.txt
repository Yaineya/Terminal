git branch - проверить наличие существующих веток
git branch [название новой ветки] - находясь в папке master ветки создать новую ветку
git checkout [название ветки] - переключиться на эту ветку
(на новой ветке работаем и делаем свои изменения,
далее в этой же ветке добавляем под гит, коммитим)
git checkout master - переходим на главную ветку
git merge branches - на главной ветке указываем какую ветку будем мерджить с главной
git push  - пушим изменения на внешний репозитрий
git push -u origin [имя ветки, которую пушим на внешний репозитрий]
